## 🚀 快速开始

### 环境要求

- Windows 7 / 10 / 11
- Visual Studio 2017 及以上
- [EasyX 图形库](https://easyx.cn/)（已安装）

### 编译运行

1. **克隆项目**
   ```bash
   git clone https://github.com/你的用户名/wing-battle.git
   cd wing-battle
   ```

2. **打开项目**
   - 用 Visual Studio 打开 `.sln` 项目文件
   - 或者直接把所有 `.cpp` 和 `.h` 文件加入空项目

3. **配置库依赖**
   - 确保项目属性 → 链接器 → 输入 → 附加依赖项中包含：
     ```
     winmm.lib
     msimg32.lib
     ws2_32.lib
     ```

4. **编译运行**
   - 选择「Release | x86」配置
   - 按 `F5` 或 `Ctrl+F5` 运行

### 联机步骤

> 📌 **两台电脑需连接同一 WiFi / 局域网**

1. **主机端**：选择「创建房间」→ 等待客机加入
2. **客机端**：选择「加入房间」→ 输入主机的局域网 IP 地址
3. 连接成功后自动开始游戏，两人一起突突突！

> 💡 **不知道主机 IP？** 按 `Win+R` → 输入 `cmd` → 输入 `ipconfig` → 找 IPv4 地址

## ⚙️ 技术实现

### 架构设计

```
┌─────────────────────────────────────────────────┐
│                     主机 (Host)                  │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐ │
│  │ 玩家1输入 │→ │ 游戏逻辑 │→ │ 状态序列化    │ │
│  └──────────┘  │ (碰撞/AI)│  │ 同步给客机    │ │
│  ┌──────────┐  └──────────┘  └───────┬───────┘ │
│  │ 玩家2输入 │         ↑              │         │
│  │ (网络接收)│         │           TCP          │
│  └──────────┘         │              │         │
└─────────────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────┐
│                     客机 (Client)                │
│  ┌──────────┐     ┌───────────────┐             │
│  │ 玩家2输入 │────→│ 发送给主机    │             │
│  └──────────┘     └───────┬───────┘             │
│                            │ TCP                 │
│                            ↓                     │
│  ┌──────────┐     ┌───────────────┐             │
│  │ 画面渲染 │←────│ 状态反序列化  │             │
│  └──────────┘     └───────────────┘             │
└─────────────────────────────────────────────────┘
```

### 核心方案：主机权威 + 状态同步

- **主机权威**：所有游戏逻辑（敌机生成、碰撞检测、计分、Boss AI）只在主机端运行，避免不同步问题
- **状态同步**：主机每帧将完整游戏状态序列化后发送给客机，客机反序列化后直接渲染
- **TCP 协议**：自定义数据包格式（包头 + 数据），内置粘包处理，保证数据可靠传输

### 网络协议设计

```
┌──────────────────────────────────────────────┐
│                 数据包格式                     │
├──────────┬──────────┬──────────┬──────────────┤
│  magic   │ dataLen  │ msgType  │    data      │
│ (2字节)  │ (2字节)  │ (1字节)  │  (N字节)     │
└──────────┴──────────┴──────────┴──────────────┘
```

**消息类型**：
- `MSG_INPUT` — 客机 → 主机：玩家按键输入（1 字节，按位存储）
- `MSG_GAME_STATE` — 主机 → 客机：游戏状态快照（包含所有实体位置、血量、分数等）
- `MSG_EVENT` — 主机 → 客机：游戏事件（过关 / 失败 / 通关）

### 数据结构

- **双向链表**：基于 Linux 内核风格的 `list_head` 实现，高效管理敌机、子弹、爆炸等动态对象
- **统一节点**：所有游戏实体共用 `Node` 结构体，通过 `type` 字段区分类型，简化内存管理
